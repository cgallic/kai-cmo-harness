# Landing Page Messaging Checklist

> Quick validation checklist for the Landing Page Messaging Workflow. Use alongside `playbooks/landing-page-messaging-workflow.md`.

---

## Phase 0: AI-Powered Persona Research

- [ ] Ran pain point discovery searches (Reddit, Twitter, forums)
- [ ] Ran competitor sentiment searches (reviews, "switched from", "alternative to")
- [ ] Ran wish list discovery searches ("wish", "should", "why doesn't")
- [ ] Ran objection mining searches ("scam", "worth it", "vs")
- [ ] Extracted 10+ verbatim quotes with sources
- [ ] Identified recurring pain point patterns (3+ mentions)
- [ ] Documented exact language and phrases used
- [ ] Saved to `clients/{client}/outputs/persona-research-ai.md`

---

## Phase 1: Persona Extraction

- [ ] Synthesized AI research into persona segments
- [ ] Matched verbatim quotes to each persona
- [ ] Identified 3-4 distinct buyer personas
- [ ] Each persona has documented:
  - [ ] Profile (one-line description)
  - [ ] Volume/value metric
  - [ ] 3+ pain points with emotional AND business impact
  - [ ] Goals (what success looks like)
  - [ ] Decision process (discovery → evaluation → trigger → objections)
  - [ ] One-sentence hook
- [ ] Created persona summary matrix
- [ ] Saved to `clients/{client}/outputs/target-personas.md`

---

## Phase 2: Cached Prediction Analysis

For each persona:
- [ ] Identified cached prediction (belief blocking action)
- [ ] Identified legitimizing label (virtue protecting inaction)
- [ ] Documented actual reality (cost of their current state)
- [ ] Wrote destabilization statement (re-indexes virtue as vice)

---

## Phase 3: Headline Generation

- [ ] Generated 10+ headlines using different tactics:
  - [ ] Re-indexing (virtue → vice)
  - [ ] Behavior attack (current action is wrong)
  - [ ] Chaos recognition (name messy reality)
  - [ ] Fear anchor (name what they fear)
  - [ ] Opportunity framing (show what they're missing)
  - [ ] Status positioning (compare to aspirational group)
  - [ ] Uncertainty attack (create doubt)
  - [ ] Question hook (ask what they wonder)
- [ ] Tagged each headline with target persona
- [ ] Scored each for memorability (1-4)
- [ ] Selected top 10 for testing

---

## Phase 4: Subheadline & CTA Generation

Subheadlines:
- [ ] Generated 6+ subheadlines with context shifts:
  - [ ] Data-driven (guessing → measuring)
  - [ ] Proof culture (hidden → visible)
  - [ ] Identity shift (user → creator)
  - [ ] Safety first (risk → security)
  - [ ] Simplicity (complex → easy)

CTAs:
- [ ] Generated 6+ CTAs with permission mechanisms:
  - [ ] Low commitment ("Your First...")
  - [ ] Zero commitment ("See How...")
  - [ ] Social proof entry ("Browse...")
  - [ ] Self-interest ("Calculate...")
  - [ ] Trivial cost ("Start with $1")
- [ ] Created combination matrix (headline + subhead + CTA)
- [ ] Saved to `clients/{client}/outputs/landing-page-test-matrix.md`

---

## Phase 5: Landing Page Variants

Created 3-4 variants:
- [ ] Trust Anchor (fear → safety)
- [ ] Monetization (revenue opportunity)
- [ ] Proof/Credibility (verification focus)
- [ ] Simplifier (organization focus)

Each variant includes:
- [ ] Hero (headline, subhead, CTAs, stats)
- [ ] Problem section (3-4 pains, attack alternatives)
- [ ] Solution section (3 value props)
- [ ] How It Works (3-4 steps, time framing)
- [ ] Social Proof (testimonials, stats)
- [ ] Calculator/Monetization (if applicable)
- [ ] Objection Handling (FAQ)
- [ ] Final CTA (repeat theme, both CTAs)

Files saved:
- [ ] `clients/{client}/outputs/landing-page-v1-trust-anchor.md`
- [ ] `clients/{client}/outputs/landing-page-v2-monetization.md`
- [ ] `clients/{client}/outputs/landing-page-v3-proof.md`
- [ ] `clients/{client}/outputs/landing-page-v4-simplifier.md`
- [ ] `clients/{client}/outputs/landing-page-prototype.html` (optional)

---

## Phase 6: Scoring & Synthesis

Four U's Scoring (each combo):
- [ ] Unique: 1-4 (can only THIS product claim this?)
- [ ] Useful: 1-4 (immediate action possible?)
- [ ] Ultra-specific: 1-4 (numbers, examples?)
- [ ] Urgent: 1-4 (reason to act today?)
- [ ] **Total: 12+/16 minimum**

Perception Engineering Scoring:
- [ ] Perception layer present (destabilizes belief): 0-3
- [ ] Context layer present (shifts norms): 0-3
- [ ] Permission layer present (removes consequences): 0-3
- [ ] **Total: 7+/9 minimum**

Final synthesis includes:
- [ ] Winning hero (headline + subhead + CTA)
- [ ] Persona-specific hooks (one per persona)
- [ ] Value props ranked by impact
- [ ] CTAs ranked by conversion potential
- [ ] Objection handling table
- [ ] Conversion formula (psychology steps)
- [ ] Traffic source mapping
- [ ] A/B test priority
- [ ] Saved to `clients/{client}/outputs/WINNING-MESSAGING-FINAL.md`

---

## Copy Quality Checks

Before finalizing:
- [ ] No generic AI phrases ("In conclusion", "It's important to note")
- [ ] No weasel words ("might", "could", "potentially")
- [ ] Specific numbers where possible
- [ ] Sounds like founder voice, not marketing speak
- [ ] Hook would stop YOU scrolling
- [ ] Every claim backed by proof or safety assurance
- [ ] CTAs are specific actions, not vague
- [ ] Objections handled BEFORE they're raised
- [ ] Mobile-friendly (short paragraphs, scannable)

---

## Validation Questions

Ask yourself:
1. Would this headline make ME stop scrolling?
2. Does the subheadline answer "what's in it for me?"
3. Is the CTA a specific action or vague?
4. Are all objections handled on the page?
5. Would a skeptic trust this?
6. Is there a reason to act TODAY?
7. Does this sound like the product, not like "marketing"?

---

## Quick Destabilization Test

For your main headline, verify:

| Check | Pass? |
|-------|-------|
| Names a behavior or belief the reader has | |
| Re-frames that behavior/belief negatively | |
| Creates dissonance (they can't unsee it) | |
| Doesn't insult - just reveals truth | |
| Makes the solution feel inevitable | |

## Traffic-to-Page Mapping

| Traffic Source | Recommended Variant | Why |
|----------------|--------------------| ----|
| Cold paid (Google, Meta) | Trust Anchor | Skeptics need safety first |
| Warm social (Twitter, Discord) | Proof or Monetization | Already have context |
| Referral / Word of mouth | Monetization | High intent, show opportunity |
| SEO / Organic search | Simplifier | Problem-aware, solution-seeking |
| Retargeting | Monetization | Already interested, push to convert |
