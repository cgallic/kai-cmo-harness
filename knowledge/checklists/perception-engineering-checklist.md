# Perception Engineering Checklist

> **Use when:** Crafting persuasive messaging, designing sales environments, handling objections, or creating any communication intended to change behavior.

---

## Pre-Work: Identify the Block

- [ ] What is the subject's **cached prediction**? (The identity label or belief driving current behavior)
- [ ] What **legitimizing label** are they using? (The "virtue" that justifies inaction: "cautious," "thorough," "perfectionist")
- [ ] What **negative outcome** are they predicting? (Status threat, judgment, failure, loss)
- [ ] What **genre** is the interaction currently in? (Exam, Boardroom, Crisis)

---

## Layer 1: PERCEPTION (Destabilize)

**Goal:** Break the cached self-model and force active processing.

- [ ] Identify the cached prediction driving the "No"
- [ ] Formulate a re-indexing statement that flips "virtue" → "vice"
  - Example: "You call it caution. It's actually a fear algorithm you didn't write."
- [ ] Use the "As If" shift if identity defense is strong
  - "What if you were someone who..."
  - "Act as if..."
- [ ] Deploy a "Magic Word" destabilizer to force attention
  - "You are obeying a rule you didn't write"
  - "What if the thing protecting you is actually limiting you?"
- [ ] Apply the Failure-to-Data reframe where relevant
  - "That's not failure, that's data acquisition"
- [ ] Verify: Does this create cognitive dissonance with current self-model?

---

## Layer 2: CONTEXT (Shift Rules)

**Goal:** Change what behaviors feel allowed without arguing.

- [ ] Audit the current Markov Blanket
  - What information is the subject seeing?
  - What options are visible vs hidden?
- [ ] Identify current genre and target genre

| From (High Stakes) | To (Low Stakes) |
|--------------------|-----------------|
| The Exam | The Lab |
| The Boardroom | The Brainstorm |
| The Courtroom | The Workshop |
| The Performance | The Practice |

- [ ] Apply genre-shifting language
  - "This isn't a final grade, it's a draft"
  - "Think of it as a beta test, not a launch"
  - "We're pinging the sensor, not submitting the thesis"
- [ ] Engineer friction appropriately
  - [ ] Add "sludge" (complexity) to unwanted path
  - [ ] Add "grease" (defaults, pre-selection) to desired path
- [ ] Constrain visible options to desired outcomes
  - Remove "Null" option from view where ethical
- [ ] Deploy social proof if appropriate
  - "This is what [similar person/company] did"
  - "Most people in your position..."

---

## Layer 3: PERMISSION (Collapse Hesitation)

**Goal:** Remove imagined penalties so action feels inevitable.

### Hypnotic Syntax (Milton Model)
- [ ] Use presuppositions: "**When** you see the results..." (not "if")
- [ ] Embed commands: "I don't know if you want to **start today** or next week"
- [ ] Apply double bind: Offer two choices that both advance goal
  - "Do you want the standard or accelerated version?"
  - Never include "or nothing" as an option

### Future Pacing
- [ ] Speak in future perfect tense
  - "When we have done this..."
  - "Six months from now, when this is running..."
- [ ] Force post-decision simulation
  - "How do you want to frame this win to your board?"
  - "What will you do with the time you save?"
- [ ] Make the decision feel already-made
  - "The price is already paid (in waiting). The only variable is whether you collect the receipt."

### Authority & Automation
- [ ] Remove the "I" from recommendations
  - "The data says..." (not "I think...")
  - "The protocol dictates..." (not "My opinion is...")
- [ ] Reference standard playbooks/precedent
  - "This is the exact framework [Authority] uses"
  - "We're not inventing anything; we're executing the standard playbook"
- [ ] Grant retroactive absolution
  - "This decision was effectively made when X happened"
  - "The circumstances have already chosen for you"

---

## Case-Specific Variations

### For Founders (Analysis Paralysis)
- [ ] Re-index "perfectionism" as "fear obedience"
- [ ] Shift from "Exam" to "Lab" genre
- [ ] Frame delay as sunk cost already paid
- [ ] "You can't debug code that isn't running"

### For Buyers (High-Ticket Hesitation)
- [ ] Re-index "waiting" as "choosing the bleed"
- [ ] Future pace past the signature to the win
- [ ] Reference competitor precedent
- [ ] "Inaction is not neutral; it's a decision"

### For Creators (Failure to Ship)
- [ ] Decouple identity from output quality
- [ ] Shift from "Library" to "River" context
- [ ] Apply double bind on regret
- [ ] "You're a factory, not a museum"

---

## Anti-Patterns (Avoid These)

- [ ] **Don't argue facts** - Shift context instead
- [ ] **Don't offer the Null option** - Exclude "or nothing" from choices
- [ ] **Don't use "I think"** - Use authoritative phrasing
- [ ] **Don't acknowledge the old frame as valid** - Re-index it as flawed
- [ ] **Don't rush** - Destabilize before attempting permission

---

## Ethical Guardrails

- [ ] Does the installed decision genuinely serve the subject?
- [ ] Would the subject endorse this approach if they understood it?
- [ ] Am I removing irrational blocks or manufacturing false urgency?
- [ ] Is there genuine informed consent beneath the technique?

---

## Quick Reference: Key Phrases

**Destabilization:**
- "You are obeying a rule you didn't write"
- "What you call X is actually Y"
- "That's not caution, that's a fear algorithm"

**Context Shift:**
- "This isn't an exam, it's a sensor ping"
- "The internet is a river, not a library"
- "We're in beta, not launch"

**Permission:**
- "When we have done this..."
- "The protocol says..."
- "The decision was already made when..."
- "Which regret do you prefer?"
